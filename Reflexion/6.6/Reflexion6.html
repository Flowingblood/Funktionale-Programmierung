Typsysteme:

In der Programmierung gibt es verschiedene Arten von Typsystemen: 1. Starke / Schwache Typisierung: Der Unterschied zwischen den beiden Typisierungen ist, dass bei der schwachen Typisierung das Typensystem umgangen werden kann. Haskell zum Beispiel ist eine stark typisierte Sprache, da das Typensystem nicht umgangen werden kann. C hingegen ist eine schwach Typisierte Programmiersprache, da jeder Pointertyp in jeden anderen Pointertyp umgewandelt werden kann. 2. Dynamische / Statische Typisierung: Bei der statische Typisierung wird einer Variable ein fester Datentyp zugeordnet. Bei der dynamischen Typisierung kann eine Variable verschiedene Datentypen annehmen. Java ist eine statisch Typisierte Programmiersprache und PHP ist eine dynamisch Typisierte Programmiersprache. 3. Explizite / Implizite Typisierung: Explizite und Implizite Typisierung gibt es nur in der statischem Typisierung. Bei der Expliziten Typisierung müssen Datentypen ausgeschrieben werden. Im Gegenteil bei der Impliziten Typisierung, hier gibt es meist ein Schlüsselwort, um die Variable zu deklarieren und der Compiler/Interpreter kann dann aus dem Kontext ermitteln um welchen Datentyp es sich handelt. 4. Optionale Typisierung: In der optionalen Typisierung werden die Typannotationen nicht benötigt aber können genutzt werden. Das ist in zum Beispiel Rust, hierbei kann der Typ angegeben werden, muss aber nicht. Es ist rein zur Lesbarkeit und Semantik annotiert.
